⚠️ Full autonomy: with CLAUDE_FLAGS=--dangerously-skip-permissions the authorized user can make the assistant run anything on your machine. The tunnel is public: whoever controls your Telegram controls the machine. Use it only if you understand the risk.

Cómo funciona

How it works

Telegram ──HTTPS──▶ r.dotrino.com/<key>/tg ──▶ [túnel] ──▶ bot ──▶ asistente IA (CLAUDE_CWD)
                    valida X-Telegram-Bot-Api-Secret-Token        memoria con --resume
  • Seguridad: valida el header X-Telegram-Bot-Api-Secret-Token → solo Telegram puede invocar tu webhook.
  • Memoria: guarda el session_id y usa --resume en cada mensaje; sobrevive reinicios.
  • Cola: si llega un mensaje mientras procesa otro, lo encola (no lo descarta).
  • Security: it validates the X-Telegram-Bot-Api-Secret-Token header → only Telegram can invoke your webhook.
  • Memory: it stores the session_id and uses --resume on each message; it survives restarts.
  • Queue: if a message arrives while it's processing another, it queues it (doesn't drop it).

pm2 save && pm2 startup

With one .env per bot you can run several (each with its own token, tunnel and folder).
